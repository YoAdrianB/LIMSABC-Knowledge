The Current Challenge: Audit Blind Spots

Current Flawed Process

1. User Saves Code

A developer edits a Parser, Rule, or Calculation in the UI.

2. Audit Log Entry Fails

Log is truncated due to size limits or the table is not audited.

3. Incomplete History

We can't see what changed, why it changed, or the full previous version.

Proposed Solutions: Two Paths Forward

Option A: Dedicated Audit Table

A new, purpose-built table in our database to store a complete history of all code changes using large-capacity fields.

Option B: Git Integration

Treating the UI-managed code as a formal codebase, committing each change to a dedicated Git repository via an automated process.
